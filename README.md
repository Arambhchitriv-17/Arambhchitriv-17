### 🏦 Project 2 — Banking Customer Engagement System

<div align="center">
<img src="https://img.shields.io/badge/Domain-Banking%20%7C%20FinTech-0055FF?style=for-the-badge"/>
&nbsp;
<img src="https://img.shields.io/badge/Duration-Dec%202025%20–%20Present-00C2FF?style=for-the-badge"/>
&nbsp;
<img src="https://img.shields.io/badge/Role-SFMC%20Developer%20%26%20Data%20Analyst-1a1a2e?style=for-the-badge"/>
</div>

<br/>

**Overview:**
Designed and managed a full omni-channel customer engagement platform for banking and credit card customers. This system handles the complete customer lifecycle — from initial onboarding through ongoing engagement and retention — using Salesforce Marketing Cloud, SQL, and Power BI. The primary focus was translating complex banking business requirements into automated campaign logic with real-time analytics and dashboards.

<table>
<tr>
<td width="50%" valign="top">

#### 📋 Campaigns Built

| Campaign Type | Channel | Trigger |
|---|---|---|
| Customer Onboarding | Email + SMS | Account creation |
| Credit Card Activation | Email | Card issued, not activated |
| EMI Reminders | SMS + Email | 3 / 7 / 1 day before due |
| Transaction Alerts | SMS | Real-time transaction event |
| Card Usage Offers | Email | Spend threshold reached |
| Retention Campaigns | Email | Inactivity / churn signal |
| Re-engagement | Email + SMS | 60-day dormancy trigger |

</td>
<td width="50%" valign="top">

#### 🛠️ Technical Implementation

**Journey Builder**
- Multi-entry journeys with decision splits based on card status, EMI due date, and transaction behaviour
- Engagement-based splits: opened → nurture path vs. unopened → SMS fallback

**SQL Segmentation**
- Filtered audiences by credit score band, outstanding balance, product type, and activation status
- Suppression logic to exclude opted-out and already-converted customers

**AMPscript Personalisation**
- Dynamic content blocks rendering personalised EMI amounts, offer values, and card-specific messaging
- Conditional content based on customer tier and product portfolio

**Power BI Dashboards**
- Campaign performance: Open Rate, CTR, Conversion Rate per campaign
- Customer lifecycle views: onboarding funnel, activation rate, retention cohorts

</td>
</tr>
</table>

<details>
<summary><b>📈 KPIs Tracked</b></summary>
<br/>

| Metric | Description |
|---|---|
| 📧 Email Open Rate | % of delivered emails opened per campaign |
| 🖱️ Click-Through Rate (CTR) | % of recipients who clicked at least one link |
| 💳 Activation Rate | % of issued cards activated within 30 days post-campaign |
| 💰 Conversion Rate | % of recipients who completed the target action (EMI paid, offer redeemed, product activated) |
| 🔄 Customer Retention Rate | % of at-risk customers retained after retention campaign |
| 📱 SMS Engagement | Delivery rate and response/click-through rate for SMS channel |

</details>

### 🏥 Project 1 — Healthcare Patient Engagement System

<div align="center">
<img src="https://img.shields.io/badge/Domain-Healthcare%20%7C%20Pharma-00AA55?style=for-the-badge"/>
&nbsp;
<img src="https://img.shields.io/badge/Duration-Sept%202025%20–%20Dec%202025-00C2FF?style=for-the-badge"/>
&nbsp;
<img src="https://img.shields.io/badge/Role-Salesforce%20Marketing%20Cloud%20Developer-1a1a2e?style=for-the-badge"/>
</div>

<br/>

**Overview:**
Developed a multi-channel patient communication and engagement system for a healthcare/pharma client using Salesforce Marketing Cloud. The system automated key touchpoints across the patient journey — appointment scheduling, medication adherence, and post-visit follow-up — reducing no-shows, improving adherence rates, and streamlining patient communication through Email and SMS.

<table>
<tr>
<td width="50%" valign="top">

#### 📋 Journeys Built

| Journey Type | Channel | Purpose |
|---|---|---|
| Appointment Booking Confirmation | Email + SMS | Confirm appointment details |
| Pre-Appointment Reminders | SMS + Email | 48hr and 2hr reminders |
| Post-Visit Follow-Up | Email | Discharge instructions, feedback |
| Medication Adherence | SMS | Daily / weekly medication reminders |
| Refill Reminders | Email + SMS | 7 days before prescription expiry |
| Missed Appointment Re-engagement | Email | Re-booking prompt |
| Patient Satisfaction Survey | Email | Post-visit NPS survey |

</td>
<td width="50%" valign="top">

#### 🛠️ Technical Implementation

**Journey Builder**
- Entry events driven by appointment system data synced via Automation Studio
- Decision splits based on appointment type, patient segment, and communication preference
- Wait activities timed to appointment datetime fields pulled from Data Extensions

**Mobile Studio (SMS)**
- SMS reminders for appointment and medication workflows
- Keyword-based reply handling for appointment confirmations (CONFIRM / CANCEL)

**AMPscript Personalisation**
- Dynamic merge fields for doctor name, appointment time, clinic location, and medication name
- Conditional content blocks for appointment type (in-person vs. telehealth)

**Automation Studio**
- Scheduled SQL queries for daily patient segment refreshes
- Suppression automation for opted-out patients and discharged records
- File import automations syncing patient data from source systems

</td>
</tr>
</table>

<details>
<summary><b>📈 KPIs Tracked</b></summary>
<br/>

| Metric | Description |
|---|---|
| 📧 Email Open Rate | % of delivered appointment and follow-up emails opened |
| 📱 SMS Engagement | Delivery success rate and patient confirmation response rate |
| 🏥 Appointment Attendance Rate | % reduction in no-show rate post-reminder automation |
| 💊 Medication Adherence Rate | % of patients confirming medication intake (reply-based tracking) |
| ⭐ Patient Satisfaction (NPS) | Net Promoter Score from post-visit survey journeys |
| 🔁 Re-booking Rate | % of missed appointment patients who rebooked via re-engagement journey |

</details>
